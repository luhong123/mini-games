## 踩坑：Edit 工具同消息批量多文件改动会"假成功"
- 同一条消息对多个文件各发 Edit，HTML 单行改动（如 `<div id="info">`）返回 Successfully 但实际未落盘（只有 tetris 的生效）
- 根因①：old_string 带 2 空格缩进，而文件实际顶格无缩进 → 不匹配但 Edit 仍返回成功（误报）；② Bash heredoc 内联中文有 locale 编码坑，old 字符串中文乱码导致 not found
- 教训：① 批量多文件 Edit 后务必 grep/Read 验证是否真落盘；② 含中文的改动优先用 Write 写 Python 脚本落盘（文件 UTF-8 可靠）；③ Read 输出 `行号→` 后第一个字符即内容起始，别把 `→` 当缩进
